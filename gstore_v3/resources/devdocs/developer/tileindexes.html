  • GET /apps/{app}/tileindexes/{uuid}/services/ogc/{wxs} Access WxS service

    Access the tile index WxS services.

    Request Parameters

    Route Parameters
    Parameter Description Type Options
    app Application alias string rgis, epscor, elseweb
    uuid Tile index identifier uuid
    wxs OGC Service Type string wms, wfs, wcs
    Query Parameters

    GSToRE uses MapServer to provide OGC services. Please see MapServer for more information regarding the required query parameters and their definitions.

    To identify which OGC services are supported for a given tile index, please refer to the tile index's service response (Get tile index description).

    Examples

    WCS Services

    The following URL specifies a particular layer name (&COVERAGE), specifies the projection (UTM Zone 11) in which BBOX coordinates are specified (&CRS), and the response projection (&RESPONSE_CRS), and returns a netCDF file (&FORMAT)

    http://129.24.63.85/apps/vwp/datasets/4e61e29b-e373-4989-923d-200748bd9a2d/services/ogc/wcs?SERVICE=WCS&VERSION=1.0.0&REQUEST=GetCoverage&COVERAGE=downwelling_longwave_flux_in_air&BBOX=569027.125,4842177.25,569452.125,4842547.25&CRS=epsg:26911&RESPONSE_CRS=epsg:26911&WIDTH=170&HEIGHT=148&Format=netcdf


    The following URL specifies a particular layer name (&COVERAGE), specifies the projection (Lat-Long NAD83) in which BBOX coordinates are specified (&CRS), and the response projection (&RESPONSE_CRS), and returns a netCDF file (&FORMAT)

    http://129.24.63.85/apps/vwp/datasets/4e61e29b-e373-4989-923d-200748bd9a2d/services/ogc/wcs?SERVICE=WCS&VERSION=1.0.0&REQUEST=GetCoverage&COVERAGE=downwelling_longwave_flux_in_air&BBOX=-116.142905556,43.7294055556,-116.137583333,43.7326972222&CRS=epsg:4269&RESPONSE_CRS=epsg:26911&WIDTH=170&HEIGHT=148&Format=netcdf


    The following URL specifies a particular layer name (&COVERAGE), specifies the projection in which BBOX coordinates are specified (&CRS), and the response projection (&RESPONSE_CRS), returns a netCDF file (&FORMAT), and which layer bands should be returned (&BANDS=1,2,3).

    http://129.24.63.85/apps/vwp/datasets/4e61e29b-e373-4989-923d-200748bd9a2d/services/ogc/wcs?SERVICE=WCS&VERSION=1.0.0&REQUEST=GetCoverage&COVERAGE=downwelling_longwave_flux_in_air&BBOX=-116.142905556,43.7294055556,-116.137583333,43.7326972222&CRS=epsg:4269&RESPONSE_CRS=epsg:4326&WIDTH=170&HEIGHT=148&Format=netcdf&BANDS=1,2,3

    Response

    The structure of the response varies depending on the OGC service and query parameters selected.
