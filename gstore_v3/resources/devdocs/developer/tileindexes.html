  • GET /apps/{app}/tileindexes/{uuid}/services/ogc/{wxs} Access WxS service

    Access the tile index WxS services.

    Request Parameters

    Route Parameters
    Parameter Description Type Options
    app Application alias string rgis, epscor, elseweb
    uuid Tile index identifier uuid
    wxs OGC Service Type string wms, wfs, wcs
    Query Parameters

    GSToRE uses MapServer to provide OGC services. Please see MapServer for more information regarding the required query parameters and their definitions.

    To identify which OGC services are supported for a given tile index, please refer to the tile index's service response (Get tile index description).

    Examples

    WCS Services

    WCS requests for layers in netCDF files will specify a single layer. Multiple bands, however, can be specified in a request using the '&Bands' directive. If no bands are specified, all bands in the netCDF layer will be returned.

    The following URL specifies a particular layer name (&COVERAGE), specifies the projection (Lat-Long NAD83) in which BBOX coordinates are specified (&CRS), and the response projection (&RESPONSE_CRS), returns a netCDF file (&FORMAT), and requests multiple bands (&Bands=1,2,3)

    http://vwp-dev.unm.edu/apps/vwp/datasets/5f080b22-1c7d-4121-b7bf-5021e14025a7/services/ogc/wcs?SERVICE=WCS&VERSION=1.0.0&REQUEST=GetCoverage&COVERAGE=downwelling_longwave_flux_in_air&BBOX=-116.142905556,43.7294055556,-116.137583333,43.7326972222&CRS=epsg:4269&RESPONSE_CRS=epsg:26911&WIDTH=170&HEIGHT=148&Format=netcdf&Bands=1,2,3

    The following URL specifies a particular layer name (&COVERAGE), specifies the projection (Lat-Long NAD83) in which BBOX coordinates are specified (&CRS), and the response projection (&RESPONSE_CRS), returns a netCDF file (&FORMAT), and requests a single band (&Bands=1)

    http://vwp-dev.unm.edu/apps/vwp/datasets/5f080b22-1c7d-4121-b7bf-5021e14025a7/services/ogc/wcs?SERVICE=WCS&VERSION=1.0.0&REQUEST=GetCoverage&COVERAGE=downwelling_longwave_flux_in_air&BBOX=-116.142905556,43.7294055556,-116.137583333,43.7326972222&CRS=epsg:4269&RESPONSE_CRS=epsg:26911&WIDTH=170&HEIGHT=148&Format=netcdf&Bands=1

    WMS Services

    WMS services will generally be for a single band per layer. As for WCS requests, this is accomplished with the '&Bands=' directive. See the examples below.

    The following URL specifies a particular layer name (&LAYERS=), specifies the projection (&SRS=4269) in which BBOX coordinates are specified, and returns a PNG file (&FORMAT=). The request does not specify which bands are to be returned, so Band 1 is returned by default.

    http://vwp-dev.unm.edu/apps/vwp/datasets/5f080b22-1c7d-4121-b7bf-5021e14025a7/services/ogc/wms?SERVICE=WMS&VERSION=1.0.0&REQUEST=GetMap&LAYERS=downwelling_longwave_flux_in_air&STYLES=&BBOX=-116.142905556,43.7294055556,-116.137583333,43.7326972222&SRS=epsg:4269&WIDTH=680&HEIGHT=592&Format=PNG

    The following URL specifies a particular layer name (&LAYERS=), specifies the projection (&SRS=4269) in which BBOX coordinates are specified, returns a PNG file (&FORMAT=), and specifies a single layer band should be returned (&BANDS=2).

    http://vwp-dev.unm.edu/apps/vwp/datasets/5f080b22-1c7d-4121-b7bf-5021e14025a7/services/ogc/wms?SERVICE=WMS&VERSION=1.0.0&REQUEST=GetMap&LAYERS=downwelling_longwave_flux_in_air&STYLES=&BBOX=-116.142905556,43.7294055556,-116.137583333,43.7326972222&SRS=epsg:4269&WIDTH=680&HEIGHT=592&Format=PNG&Bands=1,2,3
